# 安装说明（macOS Apple Silicon）

这份说明面向不熟悉终端、Python 或 MCP 的用户。正常安装只需解压和双击。

## 推荐：复制一行命令自动安装

打开 Mac 的“终端”，完整复制下面这一行并按回车：

```bash
git clone https://github.com/zyf2238070278-glitch/codex-obsidian-book-library.git && cd codex-obsidian-book-library && ./install-from-github.command
```

它会自动下载 `v0.1.0-beta.1` 全量安装包，核对 SHA-256，并安装到 `~/CodexBookLibrary`。脚本不会用不安全的“下载后直接交给 shell”方式执行网络内容；校验失败或安装目录存在但不完整时会停止。

看到“安装完成”后，完全退出并重启 Codex，然后在 Codex 中打开并信任 `~/CodexBookLibrary` 整个目录。新建任务并说“检查书库状态”。除 GitHub 授权或 macOS 安全确认外，不需要手动配置 Python、语义模型或 MCP。

下面的 ZIP 方法适合不想使用终端或需要离线保存安装包的用户。

## 安装前确认

- 当前测试版只支持 Apple Silicon Mac，不支持 Intel Mac，也不支持 Windows。
- 把 ZIP 完整解压到一个以后不会随意移动的目录，不要直接在压缩包预览里运行。
- 建议至少预留约 3 GB 空间，用于模型、Python、依赖、索引和书籍副本。
- 语义模型随安装包提供，安装时不会再单独下载模型；但首次依赖安装仍然需要联网，安装器可能需要取得 Python 和锁定版本的依赖。
- 安装时不要让 Mac 睡眠，也不要关闭安装窗口。

## 手动下载 ZIP：五步完成

1. 下载全量 ZIP，解压到一个稳定位置。以后要使用书库时保留整个目录。
2. 双击解压目录根部的 `install-macos.command`，等待安装窗口运行。
3. 看到“安装完成”后，完全退出并重启 Codex。
4. 在 Codex 中打开整个解压目录作为项目并信任项目。只有信任后，Codex 才会加载项目的 `.codex/config.toml`。
5. 新建任务并说“检查书库状态”。正常后，在 Codex 附加书籍并说“导入这本书”。

安装器默认使用 `<解压目录>/Obsidian书库`，并创建以下四个目录：

- `书库/00-待导入`
- `书库/10-原始书籍`
- `书库/20-解析文本`
- `书库/30-AI读书笔记`

对话、书籍附加和操作要求都在 Codex 中完成。Obsidian 只负责浏览这些目录中的原书、解析文本和读书笔记。

## macOS 拦截安装文件时

Gatekeeper 可能要求你确认第一次打开。只使用下面任一种系统入口：

1. 在 Finder 找到 `install-macos.command`，右键或按住 Control 点按，选择“打开”，再确认“打开”。
2. 如果刚才已被拦截，打开“系统设置”→“隐私与安全性”，找到对应提示，选择“仍要打开”。

请先确认文件来自你信任的项目发布页。不要降低整台 Mac 的安全设置，也不要照抄网上来源不明的终端命令。不同 macOS 版本的按钮位置可能略有差异，不能保证所有 Mac 都出现完全相同的提示。

## 使用已有 Obsidian Vault

如果已经有 Vault，可让安装器直接使用它。先在终端进入解压目录，再运行：

```bash
./install-macos.command --vault "<已有 Vault 的绝对路径>"
```

路径必须是已经存在或允许安装器创建内容的位置。命令中的引号不要删，尤其当目录名含空格时。安装器只会在该 Vault 下创建或使用四个 `书库/` 子目录，不会要求你把它改成 Codex 项目。

要改回默认 Vault，或更换 Vault，请重新运行不带或带有新 `--vault` 参数的安装器。然后完全退出并重启 Codex，使新配置生效。

## 安装完成后生成什么

- `.venv`：本项目的 Python 运行环境和依赖。
- `data/models`：随包语义模型的本机使用位置。
- `data/library.sqlite3`：导入状态、原文索引和语义向量数据库，书库服务首次启动或导入后出现。
- `.codex/config.toml`：指向当前解压目录、Python 和 Vault 的本机配置。
- `Obsidian书库`：未指定已有 Vault 时使用的默认 Vault。

内置模型和语义检索在本机运行。安装依赖需要联网，之后在 Codex 中生成回答也可能需要 Codex 服务；这不代表完全离线。

## 移动目录、备份与卸载

`.codex/config.toml` 保存绝对路径。移动了解压目录后，必须在新位置重新运行安装器，完全退出并重启 Codex，再打开并信任整个项目。

备份和卸载前请先阅读[隐私与数据存放](隐私与数据存放.md)。默认 Vault 位于解压目录内，删除整个目录会同时删除原书和读书笔记；请先备份需要保留的内容。

## 安装失败怎么办

安装器会显示“安装失败”及原因。常见原因是网络中断、空间不足、安装包不完整、目录不可写，或 `bin/uv` 丢失执行权限。请保留完整解压目录，处理提示中的问题后重新双击；重复运行会复用同一个 `.venv`、数据目录和 Vault。
