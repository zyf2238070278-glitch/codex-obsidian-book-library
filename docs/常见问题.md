# 常见问题

## MCP 面板里没有 `book_library`，是不是安装失败？

也可能是正常的。不同 Codex 版本的面板展示可能不同，以“检查书库状态”的结果为准。若状态可返回并显示书籍数量，书库服务已经可用。若没有结果，请确认你打开并信任的是整个解压目录，随后完全退出并重启 Codex。

## 每个新任务都要重新输入启动命令吗？

不需要。新任务不需要每次输入命令，也不用手动运行 Python 或 MCP。只要在 Codex 中打开并信任整个项目，项目配置就会随任务加载。刚移动目录、更换 Vault 或排查问题时，再说一次“检查书库状态”即可。

## 状态显示 0 本正常吗？

正常。显示 0 本表示服务已经启动，书库状态正常但还没有导入书。请在 Codex 附加支持的文件，再说“导入这本书”。

## 支持哪些格式？

当前支持 PDF、EPUB、Markdown 和 TXT。图片、Word 文档和其他格式不在当前支持范围内。

## 扫描版 PDF 为什么几乎搜不到内容？

扫描版 PDF 的页面通常只有图片，没有可提取文字。本项目不提供 OCR；请先用你信任的工具生成带文本层的 PDF，再重新导入。不要假定所有普通 PDF 都能完整提取，复杂排版也可能需要核对原页。

## 首次安装为什么比较慢？

首次安装需要联网准备 Python 和锁定版本的依赖，并创建 `.venv`。即使语义模型已经随包提供，依赖安装、校验和解压仍会花时间。请保持网络连接，预留约 3 GB 空间，并等待“安装完成”。之后重复运行通常会复用已有环境。

## Gatekeeper 阻止打开怎么办？

先确认安装包来自可信发布页。在 Finder 中右键或按住 Control 点按 `install-macos.command`，选择“打开”；或者在刚被拦截后，到“系统设置”→“隐私与安全性”选择“仍要打开”。不要降低整台 Mac 的安全设置。

## 怎么更换 Vault？

先在终端进入解压目录，再运行：

```bash
./install-macos.command --vault "<已有 Vault 的绝对路径>"
```

完成后完全退出并重启 Codex。安装器会在新 Vault 中使用四个 `书库/` 子目录，不会自动搬运旧 Vault 的原书或笔记；需要保留时请先备份并自行复制。

## 移动了解压目录后为什么失效？

项目配置记录了解压目录、虚拟环境和 Vault 的绝对路径。移动了解压目录后，请在新位置重新运行安装器，完全退出并重启 Codex，再打开并信任整个项目。

## 为什么导入后提示重复？

系统会按书籍内容识别重复导入。先说“列出书库里的书”确认已有记录；若原文件位置改变，按 Codex 返回的具体提示处理，不要反复导入同一附件。

## 安装或检索超时怎么办？

首次安装先检查网络和可用空间；检索超时则先新建任务说“检查书库状态”。如果刚完成安装或移动目录，完全退出并重启 Codex。仍失败时，保留安装窗口中的原始错误信息，重新运行安装器；不要把失败状态当作成功。
