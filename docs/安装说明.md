# 安装说明（macOS Apple Silicon）

这份说明面向不熟悉终端、Python 或 MCP 的用户。当前版本支持 macOS 16 或更高版本的 Apple Silicon（M 系列）Mac，不支持 Intel Mac，也不支持 Windows。

## 四步完成首次安装

1. 打开 Mac 的“终端”，完整复制下面这一行并按回车；它会克隆仓库并自动运行安装器：

   ```bash
   git clone https://github.com/zyf2238070278-glitch/codex-obsidian-book-library.git && cd codex-obsidian-book-library && ./install-from-github.command
   ```

2. 看到“安装完成”后，在 Codex 中打开并信任整个 `codex-obsidian-book-library` 项目。如果安装曾中断，也可以新建任务说：“请安装并检查这个书库”。
3. 完整退出并重启 Codex，再打开同一项目。
4. 重启后新建任务并说：“检查书库状态”。正常显示书籍数量后即可开始使用。

不要只打开项目里的 `Obsidian书库` 子目录。Codex 必须信任整个项目，才能读取项目规则并在重启后加载 `.codex/config.toml`。

## 安装器会准备什么

首次安装需要联网和稳定网络。安装器会下载约 500 MB 的语义模型及 Python 包，并下载经过 SHA-256 校验的固定 Node.js 与 Light OCR 运行包；实际流量取决于本机缓存。建议至少预留约 3 GB 空间，并避免安装期间让 Mac 睡眠。

项目自带固定版本的 uv 和 Apple Vision 工具。安装器会在项目内创建项目本地 Python，按锁文件安装锁定版本的依赖，验证固定语义模型，准备 RapidOCR 模型，并自动准备 Light OCR。本机无需预装 Homebrew、Python、Xcode 或 uv，也无需预装 Node.js；不应手工运行 pip、npm 或另写模型下载命令。

安装结束后，语义检索、Apple Vision、RapidOCR 和 Light OCR 都在本机运行。Codex 对话本身仍需正常联网，所以这不代表完全离线。

## 安装完成后怎么用

在 Codex 对话中附加 PDF、EPUB、Markdown 或 TXT，再说“导入这本书”。需要核对内容时说“引用原文”，回答应标注《书名》和 PDF 页、EPUB 章节或 `passage_id`；通俗解释和跨书比较也应先检索原书证据。

扫描版 PDF 导入后不会自动 OCR。只有你明确说“开始 OCR 这本书”才会启动单本任务；“查看 OCR 进度”“暂停 OCR”“继续 OCR”“处理所有待 OCR 书籍”分别用于状态、暂停、恢复和批量授权。OCR 在本机运行，原 PDF 不改写；结果可能有错字，引用前请核对 PDF 物理页。

只有你明确要求“保存”时，Codex 才会保存 AI 读书笔记。AI 笔记不是原书证据，后续引用仍需回到原书段落核验。

## 默认 Vault 与已有 Vault

默认 Vault 是 `<项目目录>/Obsidian书库`，其中会创建：

- `书库/00-待导入`
- `书库/10-原始书籍`
- `书库/20-解析文本`
- `书库/30-AI读书笔记`
- `书库/40-OCR报告`
- `书库/50-书目卡片`

`书库/书库总览.base` 会按主分类显示书籍位置、待 OCR 状态和 OCR 警告；可以直接修改卡片里的个人分类，后续同步会保留。

如果已经有 Vault，可在项目目录运行：

```bash
./install-macos.command --vault "<已有 Vault 的绝对路径>"
```

路径含空格时必须保留引号。安装器只会创建或使用书库目录，不会自动搬运另一个 Vault 的原书或笔记。更换 Vault 前请先备份需要保留的内容，完成后完整退出并重启 Codex。

## 安装完成后生成什么

- `.venv`：项目本地 Python 运行环境和依赖。
- `data/models`：固定语义模型。
- `data/ocr-models`：本机 RapidOCR 模型。
- `runtime/node` 与 `node_modules`：固定 Node.js 和 Light OCR 本机运行环境。
- `data/library.sqlite3`：导入状态、原文索引和语义向量数据库。
- `.codex/config.toml`：指向当前项目、Python 和 Vault 的本机配置。
- `Obsidian书库`：未指定已有 Vault 时使用的默认 Vault。

原书、数据库和笔记默认保留在本机；选中的少量短段落会进入 Codex 对话。问答不是零 token，也不是零内容传输。详细边界见[隐私与数据存放](隐私与数据存放.md)。

## Gatekeeper 拦截时

如果你手动运行 `install-macos.command` 时被 Gatekeeper 拦截，只使用系统提供的安全入口：

1. 在 Finder 中找到该文件，右键或按住 Control 点按，选择“打开”。
2. 或在刚被拦截后，打开“系统设置”→“隐私与安全性”，选择“仍要打开”。

不要降低整台 Mac 的安全设置，也不要运行来源不明的终端命令。

## 移动、修复、备份与卸载

移动项目目录后，在 Codex 中打开并信任新位置，再说“请安装并检查这个书库”；安装成功后完整退出并重启 Codex。重新运行会复用或修复项目环境并刷新绝对路径配置，不会删除已有书籍或笔记。

默认 Vault 位于项目目录内。删除整个项目目录进行卸载时，可能同时删除原书、解析文本、OCR 报告和读书笔记；删除、迁移或更换电脑前请先备份。

## 安装失败怎么办

安装器会显示“安装失败”及原因。常见原因是网络中断、磁盘空间不足、目录不可写、固定 uv 损坏、依赖或模型校验失败。保留完整项目目录，按错误提示修复后，在 Codex 中再次说“请安装并检查这个书库”。重复运行不会删除已有书籍或笔记。安装成功后仍要完整退出并重启 Codex，再说“检查书库状态”。
