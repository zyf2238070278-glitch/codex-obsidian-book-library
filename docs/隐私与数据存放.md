# 隐私与数据存放

这个项目把书库索引、语义模型和 Obsidian 内容放在本机，但你仍然是在 Codex 中附加文件和提问。下面分别说明“留在本机的内容”和“会进入 Codex 对话的内容”。

## 默认留在本机

原书、数据库和笔记默认保留在本机。项目不会为了建立索引而把整个书库主动上传到独立的书库服务器。

主要位置如下：

- `<解压目录>/Obsidian书库`：默认 Vault；已有 Vault 则位于安装时选择的位置。
- `书库/10-原始书籍`：导入后的原书副本。
- `书库/20-解析文本`：从书中提取的可检索 Markdown。
- `书库/30-AI读书笔记`：你明确要求保存的笔记。
- `data/library.sqlite3`：书籍状态、原文索引、语义向量和引用位置。
- `data/models`：随安装包提供的本机语义模型。
- `.venv`：本项目专用的 Python 与依赖。
- `.codex/config.toml`：本机生成的项目根目录、Python 和 Vault 绝对路径。

语义模型随安装包提供，语义检索在本机运行。安装器还会设置模型离线加载；这只描述模型检索环节，不代表完全离线。

## 会进入 Codex 的内容

- 你主动在 Codex 对话中附加的 PDF、EPUB、Markdown 或 TXT 由 Codex 处理。
- 回答书籍问题时，检索选中的少量短段落会进入 Codex 对话，供模型生成回答和标注来源。
- 你输入的问题、要求和最终回答也属于 Codex 对话内容。

本项目不会把整本书自动塞进每次问答的上下文，但问答不是零 token，也不是零内容传输。Codex 服务如何处理对话与附件，取决于你使用的 Codex 产品、账户和相关设置；请按自己的隐私要求决定是否附加敏感书籍。

## 联网边界

全量 ZIP 已内置固定版本的语义模型，检索时不需要再从模型站点下载它。首次依赖安装仍然需要联网，可能取得 Python 和锁定版本的依赖。日常在 Codex 中生成回答也可能需要 Codex 服务，因此不要把“本机检索”理解为整个问答完全离线。

## 备份建议

至少备份你的 Vault 和 `data/library.sqlite3`：

- Vault 包含原书、解析文本和已经保存的读书笔记，是最重要的用户内容。
- 数据库包含索引和导入状态；丢失后通常需要重新导入书籍。
- `data/models` 和 `.venv` 可以通过完整安装包和安装器重新准备，通常不必纳入日常备份。
- `.codex/config.toml` 含本机绝对路径，换电脑或移动目录后应重新运行安装器，而不是直接沿用旧配置。

备份前完全退出 Codex，避免数据库正在写入。恢复到新位置后重新运行安装器，再重启 Codex 并信任整个项目。

## 卸载提醒

1. 完全退出 Codex。
2. 先备份要保留的原书和笔记。
3. 如果使用默认 `Obsidian书库`，删除整个解压目录会连同 Vault、数据库和笔记一起删除。
4. 如果使用外部 Vault，删除解压目录不会自动删除外部 Vault 中的四个 `书库/` 子目录；请自行决定是否保留。

删除项目目录会移除其中的 `.venv`、`data/models`、`data/library.sqlite3` 和 `.codex/config.toml`。安装工具可能在系统用户缓存中保留下载缓存；这不包含本项目的 Obsidian 原书或读书笔记。
