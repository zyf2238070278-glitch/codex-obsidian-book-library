# 完全本地多引擎 OCR 稳定性设计

## 目标

把当前单一 Apple Vision OCR 扩展为完全本地、自动降级、可恢复的多引擎 OCR。用户仍只需在导入扫描版 PDF 后明确说一次“开始 OCR 这本书”；系统自动选择算法、修复常见页面问题、保存断点并完成索引，不要求用户理解 DPI、模型或错误类型。

系统必须保持以下边界：

- 原始 PDF 永不修改。
- OCR 运行时不联网，不上传书页，不消耗模型 API 或 Codex token。
- OCR 识别文字始终是不可信书籍证据；引用时仍按物理 PDF 页核验。
- 所有本地方法都失败的页面可以跳过，但必须记录物理页码、失败原因并在完成时报告，不得静默遗漏。
- 扫描版 PDF 导入后只标记为 `needs_ocr`；只有用户明确要求后才开始 OCR。

## 非目标

- 不承诺恢复原图中不存在的文字，也不绕过密码、DRM 或损坏文件的访问限制。
- 本阶段不提供云端 OCR 或视觉模型兜底。
- 本阶段只发布 Apple Silicon macOS 全量安装包；引擎接口需要为后续 Windows 版本保留扩展能力。
- OCR 报告不是原书证据，不能替代解析文本或物理 PDF 页。

## 用户体验

普通流程不新增技术选项：

1. 用户在 Codex 中附加书籍并导入。
2. 若书籍需要 OCR，系统只提示需要明确授权。
3. 用户说“开始 OCR 这本书”。
4. 后台逐页处理并自动选择引擎；状态只显示进度、是否有待复核页和预计剩余时间。
5. 完成后书籍照常可检索。若存在跳过页，回复简洁列出缺失的物理 PDF 页码，并提供 Obsidian OCR 报告链接。

用户无需选择 Apple Vision、RapidOCR、Tesseract、分辨率、图像增强或分块模式。

## 总体架构

### 统一页面结果

三个引擎都转换成同一种页面结果：文字行、规范化坐标、置信度、引擎名称、处理策略和诊断元数据。引擎的原始输出不得直接进入数据库或索引，必须先经过边界校验、排序和文本预算检查。

### 页面渲染与预处理

共享渲染器负责：

- 使用 PyMuPDF 按物理页渲染灰度图。
- 在长边和总像素上限内预留至少 2% 安全余量，避免整数舍入反复卡在上限。
- 按 `300 → 240 → 180 → 144 DPI` 逐级降级；实际 PDF 页面尺寸异常时直接选择安全档位。
- 自动检测页面旋转、低对比度和明显倾斜，按需生成旋转、对比度增强、二值化或降噪变体。
- 全页仍不可处理时，生成带小幅重叠的页面分块；合并时去除重叠区域的重复文字并恢复阅读顺序。
- 临时图片权限保持私有，调用结束后立即删除。

预处理不会修改原始 PDF，只生成短期存在的本机临时图像。

### 三个本地引擎

1. **Apple Vision**：默认首选。利用 macOS 原生识别，普通中文和英文页面优先走这一层。
2. **RapidOCR（ONNX Runtime）**：深度学习后备。模型随发布包固定版本提供，默认覆盖简体中文、繁体中文和英文；引擎在第一次需要时惰性加载并在当前任务内复用。
3. **Tesseract**：最后一层传统 OCR 后备。发布包内置可重定位的 arm64 运行组件以及简体中文、繁体中文、英文和方向检测数据，不依赖 Homebrew。

运行时禁止模型自动下载。缺少、损坏或哈希不匹配的模型在安装检查或任务预检阶段给出可操作错误，不在处理到一半时联网修复。

### 自动路由器

路由器按页面执行以下顺序：

1. 安全渲染标准页面变体。
2. 调用 Apple Vision。
3. 若结果合格，保存页面检查点，不再运行其他引擎。
4. 若结果报错或质量不足，使用增强变体调用 RapidOCR。
5. 若仍不合格，使用最合适的变体调用 Tesseract。
6. 若全页方案均失败，采用分块模式依次尝试 Apple Vision 和 RapidOCR；Tesseract 只在前两者仍无法产生合格结果时用于分块末级兜底。
7. 所有方案失败后保存“跳过页”检查点，继续下一物理页。

确定性错误（像素上限、无效坐标、模型缺失、无效 PDF 页面）不会原样重复三次。系统临时错误（Vision 服务暂不可用、子进程超时、短暂资源不足）最多重试两次，使用短暂递增等待，然后进入下一降级策略。

## 自动质量判定

不同引擎的置信度不可直接互相比较，因此系统使用综合质量而不是单一阈值。综合判定包含：

- 引擎返回的有效置信度。
- 坐标、Unicode 和输出结构是否有效。
- 可见字符比例、控制字符和替换字符比例、连续乱码比例。
- 文字框覆盖范围、文本密度与页面图像是否相符。
- 空白页检测；真正的空白页允许以空文本成功完成，非空页面的空结果必须进入后备引擎。
- 在低置信度情况下，两套引擎的文字数量、主要字符序列和页面分布是否相互支持。

轻微越界的识别框会被夹取到 `[0,1]`；非有限、零面积或严重越界的单个识别框会被丢弃并计数，不能因为一条坏观察而终止整页。若丢弃后有效内容不足，页面进入下一引擎。

系统保存最终选中结果及其来源，不拼接互相冲突的整页结果。只有分块模式会合并同一策略产生的非重叠区域。

## 断点、失败页和完成状态

页面检查点扩展为三种结果：

- `recognized`：有可用 OCR 文字。
- `blank`：经过页面内容检测确认的空白页。
- `skipped`：所有本地策略均失败，带有有界错误分类和尝试摘要，不保存任何伪造文字。

`recognized`、`blank` 和 `skipped` 都计入任务进度，因此单页失败不会卡住整本书。恢复任务时只处理尚无检查点的页面，不重做已成功页或已确认的跳过页；用户可以明确要求“重试 OCR 缺失页”来重新处理 `skipped` 页面。

只要至少生成一个可检索段落，书籍可以完成为可检索状态。任务同时记录 `completed_with_warnings` 语义；对外仍显示“已完成，有 N 页待复核”，避免把有缺页的任务说成完全成功。若整本书没有任何可检索文字，任务保持失败并给出更换文件建议。

## OCR 报告

每本经过 OCR 的书在 Obsidian 中生成独立报告：

`书库/40-OCR报告/<安全书名>-<book_id>-OCR报告.md`

报告包含：

- 书名、作者、`book_id`、处理时间和总物理页数。
- 成功文字页、确认空白页和跳过页数量。
- 各引擎最终采用的页面数量。
- 使用增强、降分辨率或分块恢复的页面数量。
- 每个跳过页的物理 PDF 页码、简短原因和已尝试策略。
- 可回到原始 PDF 和解析文本的 Obsidian 链接。
- “OCR 可能有识别错误，引用前请核验物理页”的固定提示。

报告不包含整页 OCR 原文，不作为 `search_books` 的索引来源，也不得写入 `书库/30-AI读书笔记`。

无跳过页时仍可生成简洁报告，便于以后排查引擎版本和页面处理情况。

## 原书一致性与性能

任务开始时对受管 PDF 做一次完整 SHA-256 校验并固定文件身份；任务结束、建立索引前再做一次完整校验。每页边界只验证设备号、inode、大小、mtime、ctime、受管路径和页数，不再每页前后重新读取整本 PDF。

页面渲染器自身使用安全打开方式并在渲染前后确认文件身份。这样保留原书变化检测，同时避免数百页大文件产生数百 GB 的重复读取。

三个引擎按需加载，任务内复用；普通页面只调用 Apple Vision。队列继续保持单本串行和页级断点，避免多个大型模型同时争用内存。

## 安装包与离线约束

macOS arm64 全量 ZIP 包含：

- 现有语义检索模型。
- Apple Vision helper。
- 固定版本 RapidOCR、ONNX Runtime 和固定哈希 OCR 模型。
- 可重定位 Tesseract/Leptonica 运行组件、必要动态库和固定语言数据。
- 第三方许可证、NOTICE、组件版本和 SHA-256 清单。

构建过程拒绝意外动态库引用、错误架构、符号链接逃逸、未列入清单的模型或二进制。安装程序在发布配置生效前执行三个引擎的合成图片自检。预计 ZIP 为 480–630 MB，安装后约 1.8–2.3 GB；最终发布说明以实际构建产物为准。

## 错误展示

普通用户只看到三类结果：

- “正在处理：X/Y 页”。
- “已完成，可以检索”。
- “已完成，可以检索；有 N 页未识别：PDF 页 …”。

只有安装不完整、PDF 加密/损坏、磁盘空间不足、整本无可检索文字或数据库无法写入时才停止并要求用户处理。详细引擎错误保存在有界本机日志和 OCR 报告中，不在对话中倾倒堆栈信息或 OCR 原文。

## 测试与验收

所有实现遵循测试先行。测试至少覆盖：

- 刚好越过总像素上限的页面不会因整数舍入卡死。
- 超大尺寸、极长页面、旋转、倾斜、低对比度、空白和分块页面。
- Apple Vision 的无效单个识别框被跳过而不是终止整页。
- Vision、RapidOCR、Tesseract 的成功、低质量、超时、模型缺失和异常输出。
- 确定性错误直接降级，临时错误按规则重试。
- 三个引擎均失败时保存 `skipped` 检查点，整本继续并生成准确页码报告。
- 暂停、崩溃、租约过期和重启后不重复成功页。
- 任务开始和结束校验整本哈希，页间不再重复读取整本文件，文件变化仍会被拒绝。
- OCR 报告不会进入书籍证据索引或 AI 笔记目录。
- 安装包在完全离线环境下通过三个引擎的合成测试，且不包含用户书籍、数据库、绝对私人路径或凭据。

真实书籍验收必须在用户再次明确授权后进行。首先恢复当前两本失败书：

- 《虚拟现实（VR）影像拍摄与制作》从物理 PDF 第 4 页继续，验证后续异常尺寸页面。
- 《世界摄影史》从物理 PDF 第 1 页重新尝试，验证异常 Vision 坐标不会终止页面。

成功标准是：正常页面自动完成；最终无法恢复的页面被准确记录并跳过；书籍仍可检索；报告页码可回到原始 PDF；原书哈希保持不变。
