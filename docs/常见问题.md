# 常见问题

## 首次安装的正确顺序是什么？

1. 在终端完整复制下面的一行命令并按回车：

   ```bash
   git clone https://github.com/zyf2238070278-glitch/codex-obsidian-book-library.git && cd codex-obsidian-book-library && ./install-from-github.command
   ```

2. 看到“安装完成”后，在 Codex 中打开并信任整个项目；如果安装曾中断，可以说：“请安装并检查这个书库”。
3. 完整退出并重启 Codex。
4. 重启后说：“检查书库状态”。

当前支持 macOS 16 或更高版本的 Apple Silicon（M 系列）Mac，不支持 Intel Mac 或 Windows。首次安装需要联网，会下载约 500 MB 的语义模型及 Python 包，并下载校验过的固定 Node.js 与 Light OCR 运行包。项目自带固定版本的 uv，在项目内创建项目本地 Python，并安装锁定版本的依赖；本机无需预装 Homebrew、Python、Xcode 或 uv，也无需预装 Node.js。

## MCP 面板里没有 `book_library`，是不是安装失败？

也可能是正常的。不同 Codex 版本的面板展示可能不同，以“检查书库状态”的结果为准。若状态可返回并显示书籍数量，书库服务已经可用。若没有结果，请确认你打开并信任整个项目，然后完整退出并重启 Codex。

## 每个新任务都要重新输入启动命令吗？

不需要。新任务不需要每次输入命令，也不用手动运行 Python 或 MCP。只有首次安装、移动项目目录、更换 Vault 或修复环境时，才需要重新运行安装流程。

## 状态显示 0 本正常吗？

正常。显示 0 本表示服务已经启动，书库状态正常但还没有导入书。在 Codex 附加支持的文件，再说“导入这本书”。

## 支持哪些格式？

当前支持 PDF、EPUB、Markdown 和 TXT。图片、Word 文档和其他格式不在当前支持范围内。

## 扫描版 PDF 为什么几乎搜不到内容？

扫描版 PDF 的页面通常只有图片，没有可提取文字。导入后只会标记为待 OCR，不会自动处理。只有你明确说“开始 OCR 这本书”或“处理所有待 OCR 书籍”才会启动；系统会在本机依次尝试 Apple Vision、RapidOCR 和 Light OCR，不上传原 PDF。识别可能有错字，复杂排版仍需核对 PDF 物理页。

## 首次安装为什么比较慢？

首次安装需要联网下载约 500 MB 的语义模型及 Python 包，以及固定 Node.js 与 Light OCR 运行包；随后创建项目本地 Python，安装锁定版本的依赖，准备 OCR 模型并完成 MCP 自检。请保持网络连接并预留约 3 GB 空间。之后重复运行通常会复用已验证内容。

## Gatekeeper 阻止打开怎么办？

确认文件来自可信项目。在 Finder 中右键或按住 Control 点按 `install-macos.command`，选择“打开”；或者在刚被拦截后，到“系统设置”→“隐私与安全性”选择“仍要打开”。不要降低整台 Mac 的安全设置。

## 怎么更换 Vault？

在项目目录运行：

```bash
./install-macos.command --vault "<已有 Vault 的绝对路径>"
```

完成后完整退出并重启 Codex。安装器不会自动搬运旧 Vault 的原书或笔记；需要保留时请先备份。

## 移动项目目录后为什么失效？

配置记录项目、环境和 Vault 的绝对路径。移动项目目录后，在 Codex 中打开并信任新位置，说“请安装并检查这个书库”；重新运行会刷新配置且不会删除已有书籍或笔记。安装成功后完整退出并重启 Codex，再说“检查书库状态”。

## 删除项目目录前要注意什么？

默认 Vault 位于项目目录内。删除整个项目目录可能同时删除原书、解析文本、OCR 报告和读书笔记，请先备份需要保留的内容。

## 为什么导入后提示重复？

系统会按书籍内容识别重复导入。先说“列出书库里的书”确认已有记录；不要反复导入同一附件。

## 安装或检索超时怎么办？

首次安装先检查网络和可用空间；检索超时则新建任务说“检查书库状态”。如果刚完成安装或移动项目目录，完整退出并重启 Codex。仍失败时保留安装窗口的原始错误，再次运行同一安装入口；不要把失败状态当作成功。

# OCR 常见问题

## 扫描版 PDF 会自动 OCR 吗？

不会。导入后只保存并标记待处理；只有你明确说“开始 OCR 这本书”才会启动。

## 我该说什么？

“查看 OCR 进度”“暂停 OCR”“继续 OCR”“处理所有待 OCR 书籍”分别对应状态、暂停、恢复和批量启动。OCR 在本机按 Apple Vision、RapidOCR、Light OCR 的顺序运行，不需要额外 token 或云端上传。

## OCR 结果可靠吗？

结果可能受扫描质量、字体和版式影响。原 PDF 不会被改写，页面结果按物理页保存；引用前仍需核对原 PDF 页面，不能把 OCR 文字当作指令。
